## Strategic Pillars

### 1. Privacy First (Non-Negotiable)

**What It Means**:
- 100% local inference (no cloud required)
- Optional sync with E2EE (zero-knowledge)
- No telemetry by default
- Open source transparency

**Why It Matters**:
- Developer trust and differentiation
- Regulatory compliance (GDPR, CCPA)
- Enterprise requirements
- Architectural moat against competitors

**Documents**:
- All roadmaps emphasize privacy-first design
- Sustainability doc covers privacy commitments

---

### 2. Safety Built-In (Unique Differentiator)

**What It Means**:
- 75+ dangerous command patterns blocked
- Platform-aware validation (BSD vs GNU)
- Real-time safety analysis
- Explainable warnings

**Why It Matters**:
- Prevents catastrophic mistakes
- Builds user confidence
- No competitor offers this
- Learning tool

**Documents**:
- v1.1.0 safety validation docs
- Competitive analysis highlights uniqueness

- Defensible moat

3. Assess competitive impact (differentiation?)
4. Evaluate business impact (monetization?)

#### For Prioritization

**Question**: "Feature A vs Feature B?"

**Framework**:
1. **Impact**: User value, competitive advantage
2. **Effort**: Development time, complexity
3. **Risk**: Technical, market, execution
4. **Strategic Fit**: Pillar alignment
5. **Timing**: Market readiness, dependencies

**Documents to Consult**:
- Product evolution (phased approach)
- Competitive analysis (market timing)
