## üì• Installation & Getting Started

### Q: How do I install Caro v1.1.0-beta?

**A**: Three installation methods:

```bash
# Method 1: Binary install script (recommended)
curl -fsSL https://caro.sh/install.sh | sh

# Method 2: From crates.io
cargo install caro@1.1.0-beta

# Method 3: Build from source
git clone https://github.com/wildcard/caro
cd caro
git checkout v1.1.0-beta
cargo build --release
```

After installation, verify:
```bash
caro --version
# Should show: caro 1.1.0-beta (...)
```

## üîí Privacy & Telemetry

### Q: What data does Caro collect?

**A**: Only **anonymous metadata**, never your commands or personal information:

**‚úÖ We collect**:
- Session timing (duration, timestamps)
- Command generation success/failure
- Platform info (OS, architecture, shell type)
- Performance metrics (generation time)
- Backend used (static matcher vs LLM)

**‚ùå We NEVER collect**:
- Your natural language input or generated commands
- File paths or directory structures
- Email addresses, usernames, or IP addresses
- Environment variables or API keys
- Hostnames (session IDs are hashed daily)

[Read full Privacy Policy](https://github.com/wildcard/caro/blob/main/docs/TELEMETRY.md)

---

### Q: Can I disable telemetry?

**A**: Yes! Multiple ways:

```bash
# Option 1: Disable in config
caro config set telemetry.enabled false

# Option 2: CLI flag (per-command)
caro --no-telemetry "find large files"

# Option 3: Environment variable
export CARO_TELEMETRY=false

# Option 4: Delete telemetry database
rm ~/.local/share/caro/telemetry.db
```

---

### Q: Can I see what data is collected before it's sent?

**A**: Absolutely! Caro is designed for **air-gapped environments** and full transparency:

```bash
# View all collected events
caro telemetry show

# Export to JSON for inspection
caro telemetry export my-data.json

# Clear all local data
caro telemetry clear
```

You can inspect every single event before sharing. Nothing is sent without your knowledge.

---

### Q: Is my data sold or shared with third parties?

**A**: **Never.** We don't sell data, period. Telemetry is used solely for:
1. Understanding usage patterns
2. Improving command generation quality
3. Prioritizing features

All data is deleted by **February 15, 2026** after analysis is complete.

---

### Q: Why opt-out instead of opt-in?

**A**: We need sufficient data to make informed product decisions. With opt-in telemetry, typically <5% of users participate, which isn't statistically meaningful.

However, we've built the most privacy-respecting opt-out system possible:
- Clear first-run disclosure
- Zero PII collection (verified by audit)
- Full transparency (inspect all data)
- Easy to disable (multiple methods)
- Automatic deletion (Feb 15, 2026)

If you're uncomfortable, disabling is simple and we respect that choice.

### Q: What commands can Caro generate?

**A**: Caro specializes in common command-line tasks:

**Supported Categories**:
- File operations (find, ls, du, stat)
- System monitoring (ps, top, lsof, df)
- Text processing (grep, sed, awk, cut)

---

### Q: Can I use Caro for dangerous operations?

**A**: Caro has built-in **safety validation** that warns you before executing potentially dangerous commands:

**Blocked by default**:
- `rm -rf /` (delete root)
- `chmod -R 777 /` (recursive permission changes)
- `dd if=/dev/zero of=/dev/sda` (disk wipes)
- `kill -9 $(ps aux | ...)` (mass process kills)

You'll see a warning and must explicitly confirm before Caro generates these commands.

## üêõ Troubleshooting

### Q: Installation fails with "permission denied"

**A**: The install script needs write access to `/usr/local/bin`:

```bash
# Option 1: Use sudo (if you trust the script)
curl -fsSL https://caro.sh/install.sh | sudo sh

# Option 2: Install to user directory
cargo install caro@1.1.0-beta
# Add ~/.cargo/bin to your PATH

# Option 3: Download binary manually
wget https://github.com/wildcard/caro/releases/download/v1.1.0-beta/caro-macos-aarch64
chmod +x caro-macos-aarch64
mv caro-macos-aarch64 ~/bin/caro
```

---

### Q: Command generation is slow (>5 seconds)

**A**: This is usually expected for complex queries using the LLM fallback. To improve:

1. **Use simpler, more direct queries**:
   - ‚ùå "I need to find files that were modified in the last week and are larger than 10MB"
   - ‚úÖ "find files modified last week larger than 10MB"

2. **Check which backend is used**:
   ```bash
   caro telemetry show | grep backend_used
   ```
   - `static`: Fast (<100ms), deterministic
   - `embedded`: Slower (1-5s), more flexible

3. **Wait for performance improvements** in v1.1.1

### Q: Telemetry commands aren't working

**A**: Check the database exists:

```bash
# Check if database exists
ls -lh ~/.local/share/caro/telemetry.db

# If missing, create it (automatic on first command generation)

# Try telemetry commands again
caro telemetry show
```

If still broken, please file an issue with:
```bash
caro --version
uname -a
ls -la ~/.local/share/caro/
```

### Q: I want to add support for my favorite command. How?

**A**: Add it to the static pattern matcher!

1. **Find the pattern file**: `src/backends/static_matcher.rs`
2. **Add your pattern** following existing examples
3. **Write tests** in `tests/beta_test_suite.rs`
4. **Submit PR** with description and test results

Example:
```rust
// Add pattern for "recent logs"
if query.contains("recent") && query.contains("log") {
    return Some("tail -f /var/log/syslog".to_string());
}
```
