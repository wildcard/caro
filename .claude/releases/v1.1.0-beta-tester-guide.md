## Beta Testing Timeline

### Day 1: Setup (Monday, Jan 13)

**Time Required**: 15-20 minutes

**Tasks**:
1. **Install Caro v1.1.0-beta**
   ```bash
   # Via install script (recommended)
   curl -fsSL https://caro.sh/install-beta.sh | sh

   # OR via cargo
   cargo install caro --version 1.1.0-beta
   ```

2. **Verify Installation**
   ```bash
   caro --version
   # Should show: caro 1.1.0-beta (...)
   ```

3. **Complete First-Run Setup**
   - Answer telemetry consent prompt
   - For beta testing, **please opt-in** (telemetry ON)
   - You can always disable it later: `caro telemetry disable`

4. **Verify Telemetry is Working**
   ```bash
   caro telemetry status
   # Should show: Status: Enabled

   caro telemetry show
   # Should show: (probably empty at this point)
   ```

5. **Report Installation Issues** (if any)
   - Platform: [macOS/Linux]
   - Architecture: [Apple Silicon/Intel/x86_64/ARM64]
   - Error message: [paste here]
   - Installation method: [script/cargo/source]

### Days 2-4: Natural Usage (Tuesday-Thursday, Jan 14-16)

**Time Required**: 10-15 minutes per day

**Tasks**:
1. **Use Caro for Real Tasks**
   - Don't try to "test" it artificially
   - Use it for your actual shell command needs
   - Try different types of queries:
     - File operations: "find large files", "list files modified today"
     - System monitoring: "show top memory processes"
     - Text processing: "search for pattern in logs"
     - Whatever you'd naturally use the shell for!

2. **Note Your Experience** (informal, mental notes)
   - What worked well?
   - What didn't work?
   - Were generated commands correct?
   - Were error messages helpful?
   - Did it feel fast/slow?

3. **Daily Export** (end of each day)
   ```bash
   # Export telemetry for privacy inspection
   caro telemetry export telemetry-day2.json
   caro telemetry export telemetry-day3.json
   caro telemetry export telemetry-day4.json
   ```

4. **Quick Privacy Check** (optional, but encouraged)
   ```bash
   # Look at the JSON to ensure it's not collecting your commands
   cat telemetry-day2.json | jq
   ```

   **What you should see**: Platform info, timing data, success/failure counts
   **What you should NOT see**: Your actual commands, file paths, environment variables

### Day 5: Feedback (Friday, Jan 17)

**Time Required**: 15-20 minutes

**Tasks**:
1. **Final Export**
   ```bash

   ```

2. **Privacy Inspection** (IMPORTANT!)
   - Review all your telemetry exports
   - Check for any personal information:
     - ‚ùå Commands or input text?
     - ‚ùå File paths or directory names?
     - ‚ùå Email addresses or usernames?
     - ‚ùå API keys or passwords?
     - ‚ùå Environment variables?

3. **Complete Feedback Survey**
   - Link: [https://forms.gle/...] (will be provided)
   - Takes ~10 minutes
   - All questions optional, but more detail helps!

4. **Send Telemetry Exports** (optional, but very helpful)
   - Email to: beta@caro.sh
   - Subject: "Beta Telemetry - [YourGitHubUsername]"
   - Attach: All telemetry-*.json files
   - Note: Only send if you're comfortable after reviewing!

## Privacy & Security

### What We're Collecting

**YES** (Metadata Only):
- Session timing (start, duration)
- Command generation events (success/failure, no content)
- Backend used (static/embedded)
- Platform info (OS, shell type)
- Error categories (no error messages)

**NO** (Never):
- Your commands or natural language input
- File paths or directory structures
- Email addresses or IP addresses
- Environment variables or API keys
- Any personally identifiable information

### Your Data Rights

- **View**: `caro telemetry show` - see everything collected
- **Export**: `caro telemetry export file.json` - get your data
- **Delete**: `caro telemetry clear` - delete local data
- **Disable**: `caro telemetry disable` - stop collection anytime
- **Opt-out**: You can withdraw from beta testing anytime

### Our Commitment

- We will NOT share your individual data with third parties
- We will ONLY use aggregate, anonymized statistics
- We will DELETE all beta data after analysis (by Feb 15)
- We will PUBLISH aggregate findings (no individual data)

Read our complete [Privacy Policy](https://github.com/wildcard/caro/blob/main/docs/TELEMETRY.md)

## Frequently Asked Questions

### Q: How much time will this take?

**A**: ~30-60 minutes total over 5 days:
- Day 1: 15-20 min (setup)
- Days 2-4: 10-15 min/day (use + export)
- Day 5: 15-20 min (feedback)

### Q: Do I need to test specific features?

**A**: No! Just use Caro naturally for your daily tasks. We want to understand real-world usage, not artificial testing.

### Q: What if I find a critical bug?

**A**: Report immediately to beta@caro.sh with subject "URGENT: Beta Critical Issue". We'll respond ASAP.

### Q: Can I keep using the beta after testing?

**A**: Yes! The beta will continue to work until GA release (Feb 15). After that, upgrade to v1.1.0 GA.

### Q: What if I'm uncomfortable with telemetry?

**A**: You can disable it anytime (`caro telemetry disable`), but we'd appreciate at least 1-2 days of data for the beta. If you're completely uncomfortable, you can still provide qualitative feedback via the survey.

### Q: Will I be credited?

**A**: Yes! With your permission, we'll acknowledge you in:
- Release announcement
- GitHub release notes
- CONTRIBUTORS.md file

### Q: What if I miss a day?

**A**: No problem! Do what you can. Even partial participation is valuable.

### Q: Can I share my experience publicly?

**A**: Yes! Feel free to blog, tweet, or discuss your beta testing experience. Please tag us so we can see it!

### Q: What happens to my feedback?

**A**: We'll:
1. Analyze all feedback (Jan 18-19)
2. Prioritize bug fixes (Jan 20-22)
3. Make go/no-go decision (Jan 23)
4. Release v1.1.0-beta or delay (Jan 24)
5. Share aggregate findings in v1.1.1 planning
