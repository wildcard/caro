## What is Caro?

**Caro** is a command-line tool that translates natural language into shell commands. Instead of memorizing complex command syntax, you simply describe what you want to do, and Caro generates the appropriate command for you.

### Example Use Cases

**File Management**:
```bash
$ caro "list files modified today"
# Generates: find . -type f -mtime 0

$ caro "find files larger than 10MB"
# Generates: find . -type f -size +10M
```

**System Monitoring**:
```bash
$ caro "show top 5 memory-consuming processes"
# Generates: ps aux | sort -k4 -rn | head -5

$ caro "check which process is using port 8080"
# Generates: lsof -i :8080
```

**Text Processing**:
```bash
$ caro "count lines in all Python files"
# Generates: find . -name "*.py" -exec wc -l {} + | tail -1

$ caro "search for TODO in JavaScript files"
# Generates: grep -rn "TODO" --include="*.js" .
```

### How It Works

Caro uses two approaches to generate commands:

1. **Static Matcher**: Fast pattern matching for common commands (86.2% success rate)
2. **LLM Backend**: AI-powered generation for complex or uncommon queries

The system automatically chooses the best approach for your query, prioritizing speed and accuracy.

## How to Report Issues

### When to Report

**Always Report**:
- ‚ùå Caro generates an incorrect command
- ‚ùå Command is unsafe or destructive without warning
- ‚ùå Platform mismatch (Linux command on macOS, or vice versa)
- ‚ùå Caro crashes or hangs
- ‚ùå Error messages are confusing or unhelpful
- ‚ùå Installation or setup problems
- ‚ùå Privacy concerns (suspicious telemetry data)

**Consider Reporting**:
- ü§î Feature works but could be better
- ü§î Unclear documentation
- ü§î Slow performance (>2 seconds for simple query)
- ü§î Unexpected behavior (even if not technically broken)

**Don't Need to Report**:
- ‚úÖ Caro works as expected
- ‚úÖ Feature requests (save for final survey)
- ‚úÖ General questions (ask in Discord instead)

## Telemetry & Privacy

### What is Telemetry?

Telemetry is **anonymous usage data** that helps us understand how Caro is being used and where it needs improvement.

**What We Collect** (if you opt in):
- ‚úÖ Commands generated (anonymized, no personal data)
- ‚úÖ Success/failure status
- ‚úÖ Performance metrics (response time)
- ‚úÖ Platform information (OS, shell type)
- ‚úÖ Error types (for debugging)

**What We NEVER Collect**:
- ‚ùå Personal information (name, email, etc.)
- ‚ùå File paths or directory names
- ‚ùå IP addresses
- ‚ùå Environment variables
- ‚ùå Your actual data or files

### Privacy Guarantee

**ZERO PII**: We have automated tests (220+) and manual audits to ensure **zero personally identifiable information** ever enters telemetry. This is a **blocking requirement** for release.

**You Control It**: Telemetry is **opt-in** and you can disable it anytime:
```bash
# Disable telemetry
caro telemetry disable

# Check status
caro telemetry status

# Re-enable later
caro telemetry enable
```

### Viewing Your Telemetry

**You can see exactly what we collect**:
```bash
# Export your telemetry data
caro telemetry export my-data.json

# Open and inspect
cat my-data.json | jq
```

**Example telemetry event** (fully anonymous):
```json
{
  "event_type": "command_generated",
  "timestamp": "2026-01-13T10:30:00Z",
  "platform": "macos",
  "shell": "zsh",
  "query_length": 25,
  "command_length": 42,
  "backend": "static",
  "success": true,
  "response_time_ms": 15
}
```

**Notice**: No personal data, no file paths, no identifiable information.

### Why Telemetry Matters for Beta

Telemetry helps us answer critical questions:
- Which command categories are most used?
- Where is Caro failing?
- Which platforms have issues?
- Is performance acceptable?

**Without telemetry**, we rely solely on manual reports, which might miss systemic issues affecting many users silently.

**Your choice**: Opt-in to help us improve Caro, or opt-out for complete privacy. Both are respected!

## Frequently Asked Questions

### General Questions

**Q: What if I can't commit 30-60 minutes every day?**

---

### Technical Questions

**Q: Does Caro require an internet connection?**

A: For the static matcher (most common queries), no. For LLM fallback (complex queries), yes. We're working on a fully offline mode for future releases.

---

**Q: Does Caro execute commands automatically?**

A: No! Caro generates the command and shows it to you. You choose whether to execute it. This is intentional for safety.

---

**Q: What shells does Caro support?**

A: Bash, Zsh, Fish. Most testing happens in Bash/Zsh, so Fish feedback is especially valuable!

---

**Q: Can Caro generate multi-line scripts?**

A: Currently, Caro generates single-line commands (or pipelines). Multi-line script generation is planned for future releases.

---

**Q: Does Caro learn from my usage?**

A: Not yet. The current version doesn't personalize based on your usage. This is a possible future feature.

---

### Privacy & Telemetry Questions

**Q: If I opt out of telemetry, can I still beta test?**

A: Yes! Telemetry is optional. However, we encourage opt-in because it helps us understand systemic issues that manual reports might miss.

---

**Q: Can I review telemetry before sending it?**

A: Yes! Run `caro telemetry export preview.json` anytime to see what we've collected. You can manually review before the final export.
