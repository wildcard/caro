## Visual Identity

### Design Principles

1. **Clean & Minimal**: Focus on content, not decoration
2. **Terminal-Inspired**: Monospace fonts, command-line aesthetic
3. **Accessible**: High contrast, readable, WCAG AA compliant
4. **Modern**: Clean, contemporary, not dated

---

### Design Elements

**Primary Aesthetic**: Terminal-inspired minimalism

**Visual References**:
- Terminal emulators (iTerm2, Alacritty)
- Command-line interfaces
- Developer tools (VS Code, GitHub)
- Technical documentation sites

**NOT**:
- Skeuomorphic (outdated)
- Overly colorful (distracting)
- Corporate/enterprise (too formal)

## Content Guidelines

### Documentation

**Style**: Clear, concise, example-driven

**Structure**:
1. Brief explanation (what)
2. Why it matters (context)
3. How to use (example)
4. Common issues (troubleshooting)

**Example** (Install Guide):
```markdown
## Installation

Install Caro via Homebrew (macOS):

```bash
brew install caro
```

**Why Homebrew**: Simple, trusted, automatically handles updates.

**Verify installation**:
```bash
caro --version
# Output: caro 1.1.0-beta
```

**Troubleshooting**: If `caro: command not found`, reload your shell: `source ~/.zshrc`
```
