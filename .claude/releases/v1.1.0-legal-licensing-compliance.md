### Why MIT License?

**Permissive and Business-Friendly**:
- Allows commercial use
- Allows modification and distribution
- Minimal restrictions
- Compatible with most other licenses

**Community Growth**:
- Encourages adoption (vs. GPL)
- Allows integration into proprietary software
- No "viral" copyleft requirements

**Simplicity**:
- Short, easy to understand
- Well-established and legally tested
- Low compliance burden

## Dependency Licensing

### Allowed Licenses

**Permissive Licenses** (‚úÖ Allowed):
- MIT
- Apache-2.0
- BSD-2-Clause, BSD-3-Clause
- ISC
- Unlicense
- CC0-1.0 (public domain)

**Weak Copyleft** (‚úÖ Allowed with restrictions):
- MPL-2.0 (Mozilla Public License 2.0)
  - Only if used as library (not modified)
- LGPL-2.1, LGPL-3.0 (Lesser GPL)
  - Only for dynamic linking (Rust crates are statically linked, avoid LGPL)

**Strong Copyleft** (‚ùå Not Allowed):
- GPL-2.0, GPL-3.0
- AGPL-3.0
- SSPL (Server Side Public License)

**Reason**: Strong copyleft licenses are incompatible with MIT and could force Caro to change license

### Developer Certificate of Origin (DCO)

**All commits must include DCO sign-off**:

```bash
git commit -s -m "feat: add new feature"
```

**DCO Text** (from Linux Foundation):
```
Developer Certificate of Origin
Version 1.1

By making a contribution to this project, I certify that:

(a) The contribution was created in whole or in part by me and I
    have the right to submit it under the open source license
    indicated in the file; or

(b) The contribution is based upon previous work that, to the best
    of my knowledge, is covered under an appropriate open source
    license and I have the right under that license to submit that
    work with modifications, whether created in whole or in part
    by me, under the same open source license (unless I am
    permitted to submit under a different license), as indicated
    in the file; or

(c) The contribution was provided directly to me by some other
    person who certified (a), (b) or (c) and I have not modified
    it.

(d) I understand and agree that this project and the contribution
    are public and that a record of the contribution (including all
    personal information I submit with it, including my sign-off) is
    maintained indefinitely and may be redistributed consistent with
    this project or the open source license(s) involved.
```

**Enforcement**: Git commit hook checks for `Signed-off-by:` line

## Privacy and Data Protection

### GDPR Compliance

**Caro collects minimal data (opt-in telemetry only)**

**GDPR Requirements**:
- ‚úÖ User consent (explicit opt-in)
- ‚úÖ Data minimization (only essential metrics)
- ‚úÖ Purpose limitation (usage analytics only)
- ‚úÖ Right to access (telemetry is local, user owns data)
- ‚úÖ Right to erasure (user can delete local telemetry)
- ‚úÖ Data portability (JSON format, easy export)

**Privacy Policy**: `docs/PRIVACY.md`

```markdown
# Caro Privacy Policy

Last Updated: 2026-01-08

## Data Collection

Caro collects minimal, anonymous usage data **only if you opt in**.

**What we collect (if you opt in)**:
- Command generation count (total, not individual commands)
- Backend usage (static matcher vs. LLM)
- Platform (macOS, Linux) and architecture (x86_64, ARM64)
- Version (e.g., 1.1.0-beta)
- Error rates (anonymized, no error messages)

**What we DON'T collect**:
- Your actual commands or queries
- Your files, directories, or system information
- Your IP address, location, or identity
- Any personally identifiable information (PII)

**How we use this data**:
- Improve command generation quality
- Prioritize platform support
- Fix bugs faster

**Data storage**:
- Stored locally on your machine (`~/.caro/telemetry/`)
- Uploaded weekly to our server (if you opt in)
- Aggregated and anonymized before analysis

**Your rights**:
- Opt out anytime: `caro config set telemetry.enabled false`
- Delete local data: `rm -rf ~/.caro/telemetry/`
- Request data deletion: Email privacy@caro-cli.dev

## Third-Party Services

We use:
- **GitHub** (hosting, issues) - [GitHub Privacy Policy](https://docs.github.com/en/site-policy/privacy-policies/github-privacy-statement)
- **Discord** (community chat) - [Discord Privacy Policy](https://discord.com/privacy)

## Contact

Questions? Email privacy@caro-cli.dev
```

## Open Source Licensing Best Practices

### License Compatibility

**MIT is compatible with**:
- Apache-2.0 (can combine, must include Apache notices)
- BSD licenses (can combine)
- GPL (MIT code can be included in GPL project, but not vice versa)

**Incompatible**:
- GPL code cannot be included in MIT project (without relicensing)

---

### Dual Licensing (Not Applicable)

**Caro uses single license (MIT)**

**Dual licensing (MIT OR Apache-2.0)** is common in Rust ecosystem but unnecessary for Caro:
- No Apache-2.0-only dependencies
- No patent concerns (Caro doesn't implement patented algorithms)
- Simplicity preferred

## DMCA and Copyright Issues

### DMCA Policy

**If we receive a DMCA takedown notice**:

1. **Review Validity**: Check if claim is legitimate
2. **Contact Contributor**: Notify contributor who added code
3. **Remove Content**: If valid, remove infringing content immediately
4. **Counter-Notice**: If invalid, file counter-notice
5. **Document**: Log incident in `LEGAL.md`

**Repeat Infringers**: Ban from project (rare, hopefully never)
