## Budget Overview

### Budget Philosophy

**Open Source Project Principles**:
1. **Minimize costs**: Leverage free/open-source tools where possible
2. **Value volunteer time**: Recognize that time is the primary cost
3. **Invest strategically**: Spend where it creates maximum impact
4. **Transparent accounting**: Track and report all expenses
5. **Sustainable**: Build processes that scale without proportional cost increase

### Workload Management

**Avoid Burnout**:
- **Max hours/week**: 20 hours for any role (part-time volunteer work)
- **Peak week buffer**: Build in slack for unexpected issues
- **Rotation**: Multiple people can cover each role (bus factor > 1)
- **Check-ins**: Weekly 1:1s to assess workload

**Red Flags**:
- Consistently working >20 hr/week
- Missing personal commitments for Caro
- Expressing burnout or frustration
- Quality of work declining

**Action**: If red flags appear, redistribute work or delay release

## Tools & Services

**Maximize Free Tiers**:
1. **GitHub**: Free for open source (unlimited public repos, CI minutes)
2. **Discord**: Free (up to 500k users before considering Boost)
3. **Vercel/Netlify**: Free tier (100GB bandwidth, sufficient for docs site)

**Why This Works**:
- Open source projects get generous free tiers
- Our scale is small (100-500 users in beta)
- We're not profit-driven (no revenue to justify paid tools)

---

### Tool Selection Criteria

**When Evaluating Tools**:
1. **Free tier available?** (mandatory)
2. **Sufficient for our scale?** (100-500 users, 1k downloads)
3. **Open source friendly?** (free for OSS or discounts)
4. **Easy to self-host?** (if we want to avoid vendor lock-in)
5. **Community support?** (active community, good docs)

**Examples**:
- ✅ **GitHub**: Free for OSS, industry standard
- ✅ **Discord**: Free, excellent community features
- ✅ **Vercel**: Free tier generous, easy deployment
- ❌ **Paid analytics**: Not needed (Grafana free tier sufficient)
- ❌ **Paid marketing**: Not needed (organic growth)

## Budget Optimization

### Cost Reduction Strategies

**1. Maximize Free Tiers**
- Always check if a free tier exists
- Open source projects often get discounts
- Use multiple free tiers (e.g., GitHub + Vercel + Cloudflare)

**2. Self-Host When Possible**
- Static site hosting: GitHub Pages (free)
- Monitoring: Self-hosted Grafana (if free tier insufficient)
- CI/CD: Self-hosted runners (if GitHub Actions too expensive)

**3. Optimize Resource Usage**
- Cache CI dependencies (faster builds, fewer minutes)
- Compress assets (reduce bandwidth)
- Lazy load features (reduce infrastructure load)

**4. Community Contributions**
- Swag donations (community members donate stickers, shirts)
- Infrastructure credits (sponsors provide cloud credits)
- Time donations (volunteers reduce paid time need)

- Potential: $10,000-50,000 (e.g., Mozilla MOSS, Prototype Fund)
- Use: Feature development, security audits, infrastructure
