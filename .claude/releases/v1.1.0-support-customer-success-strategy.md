## Support Vision

### Mission
**"Ensure every Caro user gets unstuck quickly, learns something in the process, and feels valued by our community."**

### Support Goals
1. **Fast Response**: Acknowledge every issue within 2 hours (work hours)
2. **High Resolution**: Solve >90% of issues within 24 hours
3. **Educational**: Users learn, not just get answers
4. **Community-Driven**: Empower community to help each other
5. **Continuous Improvement**: Use support data to improve product

### Support Values
- **Patient**: Never make users feel stupid
- **Thorough**: Understand root cause, don't just patch symptoms
- **Transparent**: Honest about bugs, limitations, timelines
- **Proactive**: Anticipate issues, document before users ask
- **Grateful**: Thank users for reporting issues (they're helping us improve)

## Support Principles

### 1. Users Are Helping Us, Not Bothering Us

**Mindset**: Every bug report, question, or complaint is valuable feedback.

**How We Show It**:
- Thank users for reporting issues: "Thanks for catching this!"
- Credit users in release notes: "Fixed issue reported by @username"
- Never say "This is documented" (even if it isâ€”link + apologize for not being clear)

**Good Response**:
```
Thanks for reporting this! You're rightâ€”that's a bug.

I've filed an issue: #123

We'll fix it in v1.1.1 (target: next week). I'll ping you when it's released.

Workaround for now: [temporary solution]
```

**Bad Response**:
```
This is already documented in the README. Please read the docs before asking.
```

---

### 2. Understand Before Answering

**Principle**: Ask clarifying questions before jumping to solutions.

**Troubleshooting Questions**:
1. What were you trying to do? (goal)
2. What command did you run? (action)
3. What happened? (actual result)
4. What did you expect? (expected result)
5. Platform & version? (`caro --version`, `uname -a`)

**Example**:
```
User: "Caro isn't working!"

Support (Bad): "Try reinstalling."

Support (Good):
"I'll help you troubleshoot. Can you share:
1. What command did you run?
2. What error message did you see?
3. Your platform: `uname -a`
4. Caro version: `caro --version`

This will help me pinpoint the issue."
```

---

### 3. Teach, Don't Just Tell

**Principle**: Explain the why, not just the how.

**Good Response**:
```
The issue is that `find` on macOS uses BSD syntax, which doesn't support `-printf`.

BSD alternative: find . -exec ls -l {} \;
Or use `-print0 | xargs -0 ls -l` for better handling of spaces.

Caro handles this automatically by detecting your platform.
Want to learn more about BSD vs GNU? [link to article]
```

**Bad Response**:
```
Use this command: find . -exec ls -l {} \;
```

---

### 4. Fix the Product, Not Just the Issue

**Principle**: If multiple users ask the same question, it's a product problem.

**Process**:
1. Answer the user immediately (short-term fix)
2. File a product issue (long-term fix)
3. Update documentation (prevent future questions)
4. Consider UX improvement (make it obvious)

**Example**:
```
User asks: "How do I disable telemetry?"

Response:
"Great question! Telemetry is opt-in (disabled by default).

To check status: `caro config telemetry --status`
To disable: `caro config telemetry --disable`

I'll update our README to make this more prominent. Thanks for asking!"

[Internally: File issue "Make telemetry status more discoverable"]
```

## Response Times & SLAs

### Service Level Targets

| Priority | First Response | Resolution Target | Channels |
|----------|----------------|-------------------|----------|
| **P0 Critical** | <1 hour | <24 hours | All |
| **P1 High** | <4 hours | <3 days | All |
| **P2 Medium** | <24 hours | <1 week | All |
| **P3 Low** | <48 hours | Next release | GitHub, Email |

**P0 Critical** (System Unusable):
- Caro crashes on launch
- Commands fail entirely
- Security vulnerability
- Privacy violation (PII leaked)

**P1 High** (Major Feature Broken):
- Specific category fails (e.g., all file commands fail)
- Installation broken on major platform (macOS/Linux)
- Dangerous command not blocked

**P2 Medium** (Minor Feature Broken):
- Edge case fails (e.g., large directories timeout)
- Platform-specific issue (FreeBSD only)
- Documentation incorrect

**P3 Low** (Enhancement/Question):
- Feature request
- General question
- Documentation improvement

**Step 1: Understand**
- Read issue carefully
- Check for reproduction steps
- Verify platform, version

**Step 2: Reproduce**
- Try to reproduce locally
- If can't reproduce â†’ Ask for more info
- If reproducible â†’ Confirm to user

**Step 3: Classify**
- **Type**: Bug / Feature / Question / Duplicate
- **Priority**: P0 / P1 / P2 / P3
- **Category**: Command generation / Safety / Installation / Documentation

**Step 4: Assign**
- P0: Immediate escalation to on-call engineer
- P1: Assign to relevant engineer
- P2/P3: Add to backlog, prioritize in sprint planning

**Step 5: Respond**
- Acknowledge issue
- Set expectation (timeline)
- Provide workaround if available

**Step 1: Gather Information** (5 min)
```

```

```

## Common Issues & Solutions

### Issue 1: "Caro: command not found"

**Root Cause**: Binary not in PATH

**Solution**:
```
1. Check installation:
   - Homebrew: `brew list | grep caro`
   - Manual: `ls /usr/local/bin/caro`

2. If installed but not in PATH:
   - macOS/Linux: `export PATH="/usr/local/bin:$PATH"`
   - Add to ~/.zshrc or ~/.bashrc to persist

3. Reload shell:
   - `source ~/.zshrc` (or ~/.bashrc)

4. Verify:
   - `caro --version`

Still stuck? Share your OS: `uname -a`
```

---

### Issue 2: "Generated command doesn't work"

**Root Cause**: Platform mismatch (BSD vs GNU) or inaccurate generation

**Solution**:
```
1. Check your platform: `uname -s`
   - Darwin = macOS (BSD)
   - Linux = Linux (GNU)

2. Is the command using GNU flags on macOS (or vice versa)?
   - macOS issue: Look for `--sort`, `--color=auto` (GNU-only)
   - Linux issue: Look for `-n` on `sort` without `-k` (BSD-only)

3. Test manually:
   - Copy command, run it
   - What's the error message?

4. Report to us:
   - Your query: "[natural language]"
   - Generated: "[command]"
   - Platform: `uname -a`
   - Expected: "[what you wanted]"

We'll investigate and improve accuracy!
```

---

### Issue 3: "Installation fails on Linux"

**Root Cause**: Missing dependencies (curl) or permission issues

**Solution**:
```
1. Check for curl:
   - `which curl`
   - If missing: `sudo apt install curl` (Ubuntu/Debian)

2. Try manual installation:
   - Download: `curl -LO https://github.com/[org]/caro/releases/download/v1.1.0-beta/caro-v1.1.0-beta-x86_64-unknown-linux-gnu.tar.gz`
   - Extract: `tar -xzf caro-*.tar.gz`
   - Move: `sudo mv caro /usr/local/bin/`
   - Verify: `caro --version`

3. Permission issues:
   - Use `sudo` for moving to /usr/local/bin
   - Or install to user directory: `mkdir -p ~/.local/bin && mv caro ~/.local/bin/`

Still stuck? Share your distro: `cat /etc/os-release`
```

### Content Requirements

**Every KB Article**:
- **Title**: Clear, searchable (e.g., "Fix: Caro command not found")
- **Symptoms**: What user sees/experiences
- **Cause**: Why it happens (brief)
- **Solution**: Step-by-step fix
- **Prevention**: How to avoid in future (if applicable)

**Example Article**:
```markdown
## Fix: "Caro: command not found" on macOS

### Symptoms
- Running `caro` returns "command not found"
- Caro was installed via Homebrew
- `brew list` shows caro is installed

### Cause
Homebrew binaries are not in your PATH.

### Solution
1. Add Homebrew to PATH:
   ```bash
   echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc
   ```

2. Reload shell:
   ```bash
   source ~/.zshrc
   ```

3. Verify:
   ```bash
   caro --version
   ```

### Prevention
Homebrew installation usually handles PATH automatically. If you installed
Homebrew in a non-standard location, this issue can occur.
```
