## Documentation Philosophy

### Principles

#### 1. **Show, Don't Tell**

**Use examples liberally** - users learn by doing.

```markdown
<!-- BAD: Abstract description -->
Caro converts natural language to shell commands.

<!-- GOOD: Concrete example -->
# Quick Start

Try your first command:

\`\`\`bash
$ caro "list files modified today"
find . -type f -mtime -1
\`\`\`

âœ… Safe to run! Caro validates all commands before showing them.
```

---

#### 2. **Progressive Disclosure**

**Start simple, add complexity gradually**.

```markdown
# Basic: Show the simplest form
$ caro "list files"

# Intermediate: Add options
$ caro "list files modified today"

# Advanced: Complex queries
$ caro "find large log files older than 30 days in /var/log"
```

---

#### 3. **Safety First**

**Emphasize safety features** in all documentation.

```markdown
## Safety Guarantees

Caro validates every command before showing it to you:

âœ… Blocks dangerous commands (rm -rf /, dd to raw devices)
âœ… Prevents command injection
âœ… Platform-aware (BSD on macOS, GNU on Linux)
âœ… Zero PII collection in telemetry

**You're always in control** - Caro shows commands, you decide whether to run them.
```

---

#### 4. **Platform-Aware**

**Document platform differences** clearly.

```markdown
## Platform Notes

### macOS (BSD Commands)
\`\`\`bash
$ caro "find files by size"
find . -type f -size +1M  # BSD syntax
\`\`\`

### Linux (GNU Commands)
\`\`\`bash
$ caro "find files by size"
find . -type f -size +1M  # GNU syntax (same in this case)
\`\`\`

**Caro automatically detects your platform** and generates appropriate commands.
```

---

#### 5. **Accessibility**

**Make documentation accessible to all users**.

- Clear, simple language (no jargon without explanation)
- Visual examples (screenshots, diagrams, ASCII art)
- Multiple formats (CLI, web, man pages, videos)
- Translations (community-driven)

// src/cli/tutorial.rs

pub struct Tutorial {
    name: String,
    description: String,
    steps: Vec<TutorialStep>,
}

pub struct TutorialStep {
    instruction: String,
    example_query: String,
    expected_command_pattern: Regex,
    explanation: String,
}

impl Tutorial {
    pub async fn run(&self) -> Result<()> {
        println!("ğŸ“š Tutorial: {}", self.name);
        println!("{}\n", self.description);

        for (i, step) in self.steps.iter().enumerate() {
            println!("Step {}/{}: {}", i + 1, self.steps.len(), step.instruction);
            println!("\nTry: caro \"{}\"", step.example_query);
            println!("\nPress Enter when ready...");

            // Wait for user
            let mut input = String::new();
            std::io::stdin().read_line(&mut input)?;

            // Show explanation
            println!("\n{}", step.explanation);
            println!("\n---\n");
        }

        println!("âœ… Tutorial complete!");
        Ok(())
    }
}
