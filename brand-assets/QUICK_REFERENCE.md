## Voice & Tone Rules

### DO âœ“
- Be helpful and confident
- Use technical terms accurately
- Explain safety decisions clearly
- Be witty when appropriate
- Empower users

### DON'T âœ—
- Be condescending ("actually...")
- Use corporate buzzwords
- Oversell safety ("100% safe")
- Make fun of user mistakes
- Gatekeep or be elitist

### Examples

**Good:**
```
âœ“ "This command looks dangerous. Mind if I block it?"
âœ“ "Great question! Here's why we blocked this..."
âœ“ "That'll work, but here's a safer alternative:"
```

**Bad:**
```
âœ— "ERROR: VALIDATION_FAILED_001"
âœ— "Actually, you should never run commands like that."
âœ— "I can't let you do that, Dave."
```
