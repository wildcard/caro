# Demo Playground

This is a fictional web API project used for VHS demo recordings.

## Structure

```
demo_playground/
├── src/                 # Rust source code
│   ├── main.rs         # Main server
│   ├── models.rs       # Data models
│   └── database.rs     # Database layer
├── config/             # Configuration files
│   ├── nginx.conf      # Nginx config
│   └── .env           # Environment variables
├── scripts/            # Utility scripts
│   ├── migrate.sh      # Database migrations
│   └── deploy.sh       # Deployment script
├── logs/               # Log files
│   ├── app.log         # Application logs
│   └── access.log      # Access logs
├── docs/               # Documentation
├── tests/              # Test files
├── Cargo.toml          # Rust dependencies
└── README.md           # Project documentation
```

## Purpose

This directory provides a realistic project structure for VHS demos, so that
commands like `ls`, `find`, `grep`, etc. show meaningful, professional output
instead of just tape files.

## Usage in VHS

In tape files, change to this directory first:

```
Hide
Type "cd demo_playground"
Enter
Type "clear"
Enter
Show
```

Then run commands that will show this nice project structure.
