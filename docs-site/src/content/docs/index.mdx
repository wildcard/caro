## Quick Start

<Tabs>
  <TabItem label="Install">
```bash
# Install via cargo
cargo install caro

# Or with Homebrew (macOS)
brew install wildcard/tap/caro
```
  </TabItem>
  <TabItem label="Usage">
```bash
# Convert natural language to shell commands
caro "find all rust files modified in the last week"
# → find . -name "*.rs" -mtime -7

caro "show disk usage sorted by size"
# → du -sh * | sort -hr

caro "count lines in all python files"
# → find . -name "*.py" -exec wc -l {} + | tail -1
```
  </TabItem>
  <TabItem label="Configure">
```bash
# Set your preferred backend
export CARO_BACKEND=mlx    # Apple Silicon
export CARO_BACKEND=ollama # Any platform

# Optional: customize model
export CARO_MODEL=codellama
```
  </TabItem>
</Tabs>

## Why caro?

<CardGrid stagger>
  <Card title="No Cloud Dependency" icon="laptop">
    Everything runs locally. No API keys, no rate limits, no privacy concerns.
    Works offline after the initial model download.
  </Card>
  <Card title="Single Binary" icon="seti:rust">
    One Rust binary under 50MB. No Python environments, no containers,
    no complex dependencies to manage.
  </Card>
  <Card title="Multiple Backends" icon="setting">
    Supports MLX for Apple Silicon, Ollama for local inference,
    and vLLM for high-performance serving.
  </Card>
  <Card title="Developer Friendly" icon="open-book">
    Intuitive CLI with colored output, confirmation prompts,
    and helpful error messages.
  </Card>
</CardGrid>
