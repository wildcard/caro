# Work Packages: Installation and Setup Documentation

**Inputs**: Design documents from `/kitty-specs/008-installation-and-setup/`
**Prerequisites**: plan.md (implementation architecture), spec.md (3 user stories), research.md (technical decisions), data-model.md (entities), quickstart.md (developer guide)

**Tests**: Manual testing of page rendering, responsive behavior, link validation, and accessibility. No automated tests requested by stakeholders.

**Organization**: Fine-grained subtasks (`Txxx`) roll up into work packages (`WPxx`). Each work package is independently deliverable and testable per user story priorities (P1: Quick Start, P2: Installation, P3: Setup).

**Prompt Files**: Each work package references a matching prompt file in `/tasks/planned/` generated by `/spec-kitty.tasks`. This file provides the high-level checklist; detailed implementation guidance is in the prompt files.

## Subtask Format: `[Txxx] [P?] Description`
- **[P]** indicates the subtask can proceed in parallel (different files/components).
- File paths relative to worktree root (e.g., `website/src/pages/quick-start.astro`).

## Path Conventions
- **Website pages**: `website/src/pages/`
- **Components**: `website/src/components/` (existing), `website/src/components/docs/` (new)
- **Layouts**: `website/src/layouts/` (existing, reuse Layout.astro)
- **Reference content**: `README.md`, `website/src/components/Download.astro`

## Work Package WP01: Foundation - Shared Documentation Components (Priority: P0)

**Goal**: Create reusable Astro components for all documentation pages (CodeBlock, PlatformTabs, InstallMethod, ComingSoonBadge).
**Independent Test**: Components render correctly in isolation; can be imported and used in any page.
**Prompt**: `/tasks/planned/WP01-foundation-components.md`

### Included Subtasks
- [ ] T001 Create `website/src/components/docs/` directory
- [ ] T002 [P] Implement `CodeBlock.astro` with copy-to-clipboard button and syntax highlighting
- [ ] T003 [P] Implement `PlatformTabs.astro` with localStorage persistence and responsive tabs
- [ ] T004 [P] Implement `InstallMethod.astro` card component with difficulty badges
- [ ] T005 [P] Implement `ComingSoonBadge.astro` with GitHub issue linking

### Implementation Notes
- All components should match existing website design system (orange gradient, typography, spacing)
- CodeBlock uses clipboard API or library for copy functionality
- PlatformTabs should stack on mobile, display tabs on desktop
- InstallMethod shows collapsible Prerequisites and Verification sections
- ComingSoonBadge should be visually distinct (orange/yellow) and link to GitHub issues

### Parallel Opportunities
- T002-T005 are fully parallelizable (independent component files)

### Dependencies
- None (foundation work).

### Risks & Mitigations
- **Copy button browser compatibility**: Use feature detection for clipboard API; provide fallback for older browsers.
- **Design consistency**: Reference existing components (Navigation.astro, Download.astro) for color/spacing patterns.

## Work Package WP02: Quick Start Page - MVP (Priority: P1)

**Goal**: Create Quick Start Guide page enabling first-time users to install and generate their first command in under 5 minutes.
**Independent Test**: New user can follow guide on clean system and successfully run first Caro command within 5 minutes.
**Prompt**: `/tasks/planned/WP02-quick-start-page.md`

### Included Subtasks
- [ ] T006 Create `website/src/pages/quick-start.astro` with Layout.astro structure
- [ ] T007 Write Step 1: Install section (automated script OR cargo install)
- [ ] T008 Write Step 2: Verify installation (run `caro --version`)
- [ ] T009 Write Step 3: Generate first command example (e.g., "list all files")
- [ ] T010 Write Step 4: Next steps (links to installation.astro and setup.astro)

### Implementation Notes
- Use CodeBlock component for all commands
- Keep instructions minimal - no long explanations
- Include expected output for verification step
- Total page reading time should be < 3 minutes
- Use sequential numbered steps (1-4)

### Parallel Opportunities
- Content writing (T007-T010) can be drafted in parallel once page structure exists

### Dependencies
- Depends on WP01 (requires CodeBlock component).

### Risks & Mitigations
- **Automated script availability**: If script doesn't exist, use "cargo install caro" as primary method and note automated script as "Coming Soon".
- **Backend confusion**: Clearly state which backend is used by default (likely Ollama or vLLM) to avoid user confusion.
