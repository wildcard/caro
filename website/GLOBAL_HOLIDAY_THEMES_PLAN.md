# Global Holiday Themes Plan

## Overview

This document outlines a culturally inclusive approach to holiday themes for the Caro website. Rather than defaulting to any single holiday, all themes are **opt-in**, respecting users from all backgrounds and cultures.

## Current Implementation (v1.0)

- **Christmas** (December) - Western Christian tradition
- **Hanukkah** (December, varies) - Jewish tradition
- **New Year** (January 1) - Gregorian calendar celebration

All themes are now **opt-in by default** as of the latest update.

---

## Feature Development Process

> **Important**: All new holiday themes MUST follow PRD-first development:
> 1. Create a PRD in `docs/prds/holidays/`
> 2. Get PRD approval
> 3. Use Spec-Kitty or Spec-Kit workflow for implementation
> 4. Follow cultural sensitivity guidelines before launch

## PRD Template for Holiday Themes

Each holiday theme requires a PRD before implementation:

```markdown
# PRD: [Holiday Name] Theme

## Overview
- **Holiday**: [Name]
- **Region/Culture**: [Primary regions]
- **Estimated Users**: [Number affected]
- **Target Release**: [Quarter/Year]

## Cultural Research
- [ ] Consulted cultural experts
- [ ] Verified authentic symbols
- [ ] Confirmed appropriate colors
- [ ] Reviewed religious sensitivities

## Design Specifications
- **Primary Color**: #XXXXXX
- **Secondary Color**: #XXXXXX
- **Accent Color**: #XXXXXX
- **Symbols**: [List of appropriate symbols]
- **Effects**: [Proposed visual effects]

## Date Detection
- **Start Date**: [Formula or fixed date]
- **End Date**: [Formula or fixed date]
- **Calendar Type**: [Gregorian/Lunar/Islamic/etc.]

## User Stories
1. As a user from [culture], I want to see [holiday] theme during [dates]
2. As a user, I want to easily enable/disable holiday themes
3. As a user with vestibular disorders, I want to disable motion effects

## Acceptance Criteria
- [ ] Theme colors meet WCAG AA contrast
- [ ] Effects respect prefers-reduced-motion
- [ ] Toggle works on mobile and desktop
- [ ] Date detection is accurate
- [ ] Cultural review approved

## Success Metrics
- Theme adoption rate
- User feedback sentiment
- Accessibility complaints (target: 0)
```

## Technical Implementation Guidelines

### Date Detection Library Requirements

```javascript
// Required calendar systems
const CALENDAR_SYSTEMS = {
  gregorian: 'Intl.DateTimeFormat',
  chinese: 'chinese-lunar-calendar',
  hebrew: 'hebcal',
  islamic: 'hijri-converter',
  hindu: 'panchang',
  persian: 'jalaali-js'
};
```

### Theme Structure (per holiday)

```css
.[holiday-name] {
  --holiday-primary: #XXXXXX;
  --holiday-secondary: #XXXXXX;
  --holiday-accent: #XXXXXX;
  --holiday-bg: #XXXXXX;
}
```

### Effect Components Needed

1. **Universal Effects** (reusable)
   - Confetti (configurable colors)
   - Fireworks
   - Sparkle/glow
   - Falling particles (snow, petals, etc.)

2. **Culture-Specific Effects**
   - Lanterns (Chinese/Japanese)
   - Diyas/lamps (Indian)
   - Rangoli patterns (Indian)
   - Flower petals (Japanese sakura)
   - Color splash (Holi)

---

## User Experience Principles

1. **Opt-in Only**: Never auto-apply any holiday theme
2. **Easy Discovery**: Show seasonal toggle when holidays are approaching
3. **Respectful Representation**: Research each holiday's significance
4. **Performance**: Effects toggleable, respect reduced-motion
5. **Accessibility**: WCAG AA color contrast minimum
6. **Inclusive Naming**: Use culturally accurate names

---

## Cultural Sensitivity Guidelines

1. **Research**: Consult people from each culture before implementation
2. **Avoid Stereotypes**: Use authentic symbols only
3. **Religious Respect**: Distinguish religious vs cultural aspects
4. **Naming**: Use proper names (e.g., "Lunar New Year" not "Chinese New Year")
5. **Timing**: Respect actual celebration dates
6. **Opt-out**: Always provide easy theme disable
